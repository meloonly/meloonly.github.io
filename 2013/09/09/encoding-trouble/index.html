那些被编码坑过的往事

发表于 2013-09-09

写在最前面，最简洁地描述编码和解码

编码：字符char[]->二进制byte[]（按照某种编码规则）
解码：二进制byte[]->字符char[]（按照某种编码规则）

最近一次操作失误导致网站提交乱码，研究了一下资料，直接原因是新增加的那个过滤器放置在了CharacterEncodingFilter之前了。也就是未执行request.setCharacterEncoding之前，去获取了request的parameter，而这个操作导致了网站post提交的数据没有按照指定的编码格式解码。
查看了servlet api里对HttpServletRequest setCharacterEncoding的解释：

Overrides the name of the character encoding used in the body of this request. This method must be called prior to reading request parameters or reading input using getReader().

意思就是说：

  • 这个方法首先只对请求体里面的数据有效果，故对get方式提交的数据没有作用。
  • 其次这个方法必须在读取请求参数之前调用。

而我之前的步骤是在此方法执行之前就获取了参数，此时请求里的参数已经被WEB容器的默认编码decode完了，之后再到那个过滤器指定编码已经无效果，于是就导致了乱码。
